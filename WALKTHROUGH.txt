PHASE 2.1: AUTH PIPELINE FIXES
-------------------------------

1. Google OAuth "provider not enabled" error
   - Cause: Google provider wasn't toggled on in Supabase dashboard
   - Fix: Enable at Authentication → Providers in Supabase project
   - Also need Google Cloud Console OAuth credentials (Client ID + Secret)

2. Auth callback redirect loop
   - Symptom: Google sign-in worked (user created) but browser landed back
     at / instead of /library
   - Root cause: src/app/auth/callback/route.ts was setting session cookies
     on an intermediate NextResponse.next() but returning a bare
     NextResponse.redirect() — cookies dropped, middleware saw no session,
     kicked user back to /
   - Fix: Create the redirect response FIRST, let exchangeCodeForSession
     write cookies directly onto it via the setAll callback
